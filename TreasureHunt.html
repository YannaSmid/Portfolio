Treasure Hunt

A Game based on Sound, Space, and Interaction.

Team Members: Nikki Rademaker, Yanna Smid

Project Overview

Step into the invisible world of the Treasure Hunt, an immersive and interactive auditory game where players rely on their ears to locate hidden treasure. But beware, lurking zombies guard the treasure, and a ticking timer adds to the pressure. Can you find the treasure before time runs out, without falling prey to the zombies?

The Treasure Hunt combines sound, space, and interaction to create a unique sensory experience. Designed for the Sound, Space, Interaction course, this project utilizes PureData and the HTC Vive VR set to transform a physical space into an engaging auditory treasure hunt.

Features

  • Interactive Gameplay: Players can navigate through a room with a tracker in their hands, searching for the treasure guided solely by sound. The tracker’s position determines the proximity to the treasure and zombies, allowing the physical engagement of the game.
  • Randomization: Treasure and zombie locations are randomly assigned in the room but remain sufficiently spaced to ensure gameplay balance. The time limit is also randomly selected to add variability. This ensures a different position of the game each playthrough.
  • Game Experience: Zombies “guard” the treasure with growling audio cues getting louder when approaching the zombies. This significantly affects the intensity of the game. Physical obstacles in the room heighten the difficulty, to manoeuvre through the room.

Technical Details

The system keeps track of the player's location through the HTC Vive basestations. The location data is sent to a PureData patch through Unity. The following hardware and software are required:

  • HTC Vive VR Set: The HTC Vive tracker is used to track the player’s position and orientation, relaying real-time x and y coordinates to the game system.
  • Speakers: Four speakers are placed in the room, one in each corner, to enhance spatial sound, immersing players in the auditory experience.
  • PureData: Processes sound cues, using oscillators and synthesized noise for the tracker and zombies.
  • Unity: Handles spatial integration, connecting the tracker data to the PureData patch.

Design Features

The system keeps track of the player's location through the HTC Vive basestations. The location data is sent to a PureData patch through Unity. The following hardware and software are required:

  • Beeping Sound from the Tracker: We synthesized a beeping sound through an oscillator in PureData, starting at 380 Hz. The beep frequency increases as the player nears the treasure.
  • Zombie Growls: A samplified .wav file from Minecraft is used for zombie growls. We amplified the sound proportionally to the player’s proximity to the zombies.
  • Timer: Ticking sound starts softly, progressively intensifying until time runs out. Volume scales from 0.5 to 13 over the randomly assigned time limit.
  • • Game End Feedback: A victory sound is played when the player has found the treasure on time. A defeated game over tune is played when time runs out. An evil laughter sound is played when the player was captured by the zombies.

Process

  • 1. Conceptualization: My teammate Nikki Rademaker and I are interested in Game Development and we quickly came to the conclusion that we wanted to create a fun interactive game using the environment and intriguing sound.
  • Set-up: The HTC Vive base stations needed to be installed in the room to the PC and Computer screen. After everything was set up, the HTC Vive application had to be installed on the PC, and the virtual room had to be assigned through the application. Moreover, four speakers were placed in the room, used as output mechanism of the PC.
  • PureData Unity connection: A Unity project was developed to connect to PureData to transfer the data tracked by the HTC Vive tracker to the PureData patch.
  • Implementation: The PureData patch was implemented and ran on the PC connected to the HTC Vive application. This patch contained all the necessary code for the space and sound interaction.
  • Finetuning: After the implementation was done, the whole set-up and program was tested. Through multiple test runs, the sound parameters such as frequency and amplitude, were adjusted to allow optimal sound experience.
  • Showcasing: The program was showcased and played by multiple friends and classmates.

Reflection

Working on Treasure Hunt was a rewarding experience that allowed us to combine our technical knowledge with creative problem-solving. We had fun to create a different type of game where a player is physically emerged in the game, allowing to use their surroundings. We successfully integrated the HTC Vive set for precise tracking and spatial interaction and created a compelling auditory experience that challenged players’ spatial awareness and listening skills. Moreover, we designed a dynamic, unpredictable gameplay loop that kept players engaged. We learned that the integrating Unity with PlugData is possible using Open Sound Control making more projects possible using real-time sound and space in Unity. Players were very positive about the game and they were insisting to try it again after they did not manage to find the treasure.
